1. Link 비유: 클릭하면 다른 방으로 순간 이동시켜주는 **문(Door)**입니다.
설명: 원래 웹사이트에서 다른 페이지로 갈 때는 <a> 태그를 썼지만, 리액트에서는 페이지 전체를 새로고침하지 않고 부드럽게 화면만 갈아끼우기 위해 Link를 사용합니다.

2. nav 비유: 식당 입구에 있는 메뉴판 모음입니다.
설명: HTML 태그 중 하나로, "여기는 내비게이션(이동 메뉴)들이 모여 있는 곳이에요"라고 컴퓨터에게 알려주는 역할을 합니다. 디자인적으로는 큰 차이가 없지만 의미를 명확히 해줍니다.
3. hr 비유: 요리가 섞이지 않게 그릇 사이에 놓는 칸막이입니다.
설명: Horizontal Rule의 약자로, 화면에 가로로 수평선을 하나 쭉 그어줍니다.

4. Outlet **"자식 요리가 담길 빈 접시"**입니다.
설명: 10번 질문의 Contact 코드 안에 <Outlet/>이 있죠? 이건 "서울지점"이나 "부산지점"을 눌렀을 때, 그 상세 내용이 **이 자리(Outlet 위치)**에 나타나게 해달라는 표시입니다.

- 여기서 element의 역할, 여기서 *의 역할, 여기서 /가 있는곳도 있고 없는곳도있음 이 의미, 여기서 왜  <Route path='contact' element={<Contact/>}> 이건 path 서울,부산을 감싼뒤에 태그를 닫았는지,
element의 역할: 주소창에 특정 주소를 쳤을 때 **"실제로 보여줄 컴포넌트(화면)"**가 무엇인지 지정하는 연결고리입니다.

(별표)의 역할: **"나머지 전부"**라는 뜻입니다. 위에서 정해준 주소 외에 엉뚱한 주소(예: /abc)를 입력하면 전부 PageNotFound 화면을 보여주라는 뜻이에요. (일종의 망태기 같은 역할이죠!)

/가 있고 없고의 차이: * /가 있으면: "우리 마을 입구(메인 주소)부터" 시작하는 절대적인 주소입니다.

/가 없으면: "현재 내가 있는 곳에서부터" 이어지는 상대적인 주소입니다.

왜 감싸서 닫았나요? (중첩 라우팅): Contact 안에 서울과 부산이 살고 있기 때문입니다.

contact/seoul이나 contact/busan처럼 **공통된 주소(contact)**를 공유하고, 화면 안의 특정 부분만 갈아끼우고 싶을 때 이렇게 감싸서 작성합니다.

6. BrowserRouter, Routes, Route
- BrowserRouter (내비게이션 장치): 우리 앱 전체에서 "길 찾기 기능"을 쓸 수 있게 해주는 가장 큰 울타리입니다. (이게 없으면 길을 못 찾아요!)
- Routes (표지판 모음집): 여러 가지 길(Route)들을 하나로 묶어주는 리스트입니다.
- Route (개별 표지판): "A 주소로 오면 A 화면을 보여줘"라고 적힌 하나하나의 규칙입니다.

answer
1. params (파라미터/매개변수)
비유: 배달 기사님께 전달하는 **'상세 메모'**입니다.

설명: 함수나 컴포넌트가 실행될 때 필요한 정보들을 담고 있는 보따리예요. 9번 예시에서 params.row는 클릭한 "그 줄(row)"의 자동차 정보를 몽땅 담고 있는 메모지라고 생각하면 됩니다.

2. useMutation (데이터 변경하기)
비유: '주문서 넣기, 수정하기, 삭제하기' 같은 행동 그 자체입니다.

설명: 서버의 데이터를 바꿀 때(추가/수정/삭제) 사용하는 도구입니다. "이 버튼 누르면 서버에 있는 차 정보 지워줘!" 같은 명령을 내릴 때 쓰죠.

3. useQuery (데이터 가져오기)
비유: **'진열장 구경하기'**입니다.

설명: 서버에서 데이터를 가져올 때 사용하는 도구입니다. 화면이 켜지자마자 "차 목록 좀 보여줘!" 하고 데이터를 요청해서 가져오는 역할을 합니다.

4. DataGrid 관련 (표 만들기)
DataGrid: 엑셀 같은 **'똑똑한 표 전체'**입니다.

GridColDef: **'표의 기둥(컬럼) 정의서'**입니다. "첫 번째 칸은 브랜드, 두 번째 칸은 가격으로 할 거야"라고 규칙을 정하는 거예요.

GridCellParams: 표의 **'칸 하나(Cell)'**에 들어있는 상세 정보들입니다. "이 칸은 어떤 줄에 있고, 값은 뭐지?" 같은 정보를 담고 있어요.

5. Snackbar (잠깐 뜨는 알림창)
비유: '토스트기에서 툭 튀어나오는 빵' 같은 알림입니다.

설명: 화면 하단에 "저장되었습니다" 하고 잠깐 나타났다가 사라지는 작은 메시지 창이에요.

6. filterable (필터 가능여부)
설명: 표에서 특정 조건(예: '현대' 차만 보기)으로 걸러내는 기능을 켤지 말지 정하는 거예요. true면 깔대기 모양 아이콘이 생겨요.

7. sortable (정렬 가능여부)
설명: 클릭했을 때 가나다순이나 숫자 순서대로 정렬할 수 있게 할지 정하는 거예요.

8. disableColumnMenu (컬럼 메뉴 끄기)
설명: 표 머리글(Column) 옆에 점 세 개 모양 메뉴 버튼이 있죠? 그걸 안 보이게 숨기는 설정입니다. 화면을 깔끔하게 만들 때 써요.

9. 코드 분석: 삭제 확인창 띄우기
TypeScript
if (confirm(`${params.row.brand}... 삭제하시겠습니까?`)) { 
  mutate(params.row._links.self.href) 
}
해설: 1. 사용자가 삭제 버튼을 누르면 브라우저 기본 알림창(confirm)이 뜹니다.
2. "예(확인)"를 누르면 if문 안으로 들어갑니다.
3. mutate(...)가 실행되면서, 해당 자동차의 고유 주소(href)를 서버로 보내 "삭제해줘!"라고 요청하는 과정입니다.

10. autoHideDuration (자동 숨김 시간)
비유: **'메시지가 사라지는 타이머'**입니다.

설명: 아까 말한 5번(Snackbar)이 몇 초 뒤에 자동으로 닫힐지 정하는 숫자예요. 단위는 밀리초(ms)라 3000이라고 쓰면 3초 뒤에 쓱 사라집니다.
