7. 버튼클릭시에 get요청이 가야한다. 키워드를 포함해서 
현재 input형태의 onChange이벤트에 대해서 arrow function을 썼지만 button 의 onclick이벤트의 경우 axios.get을 해야하고 then/catch등 체이닝메서드가 요구되기 때문에 onclick={} 는 좀 비효율적이다.
그래서 핸들클릭이라고하는 별개의함수를 정의하고 onclick = {handclick}으로 처리.

```tsx
import { useState } from 'react'
import axios from 'axios'
import './App.css'

type Repository = {
  id: number;//고유값필요 인덱스로 표기, 순서바뀌면 고유아니기에.. db에 id로 Pk 나중에 map()을적용했을때 사용
  full_name: string;
  html_url: string;
};

function App() {
  const [keyword, setKeyword] = useState('');
  const [repodata, setRepodata] =useState<Repository[]>([]); // 제네릭형식으로 Repository넣기 
  const handleClick = () =>{ 
    axios.get<{items: Repository[] }>(`http://api.github.com/search/repositories?q=${keyword}`)
      .then(response => setRepodata(response.data.items))
      .catch(error => console.log(error));
  
  }

  return (
    <>
    <input type="text" onChange={e => setKeyword(e.target.value)} value={keyword} 
    />
    <button onClick={handleClick}>검색</button>
    {repodata.length === 0 ?(
      <p>검색결과가 없음.</p>

    ):(
      <table>
        <tbody>
          {repodata.map(repo => (
            <tr key={repo.id}>
              <td>{repo.full_name}</td>
              <td>
                <a href={repo.html_url}>{repo.html_url}의 리포지토리</a>
              </td>
            </tr>
          ))}
        </tbody>
      </table>
    )
    }
    </>
  )
}

export default App

```

## useQuery
```jsx
import {useQuery} from `@tanstack/react-query`
//유즈쿼리는 객체이다 
const {data, isLoading, error} = useQuery(queryKey, queryFn, option);

```
- use Query의 return 값은 이 세가지 말고도 여러값이있다.
자주 사용하는 것 세가지 queryKey, queryFn, option

- useQuery 는 **비동기 통신을 통해 get요청으로 데이터를 불러오는데사용**
isloading은 비동기자료형 boolean형이고 
true, false 로 바뀌고 `data`에 요청한 데이터가 담기게 된다. 
`error`는 오류발생시 리턴.

## useQuery에 들어가는 argument
1. `querykey`: useQuery마다 부여되는 고유 key 단순문자열을 할 수 있고 배열로도가능
2. `queryFn` : 쿼리가 데이터를 요청하는데 사용하는함수 
-- fetch나 axios의 리턴값의 자료형은?(비동기작업의결과물)promise리턴값 이다.
 fetch api를 사용한다면 response.json()을 통해서 json형태로 바꿔주고 axios를 쓴다면 내부적으로 알아서 json으로 바꿔줌.

3. `options` : 생략가능한 argument로 옵션 추가시에 사용된다. enable,retry 등이잇다.

# useMutation
- 형식
```jsx
import {useMutate} from '';

const {data, isLoading, mutate} = userMutation(mutation, options);

```
- useQuery 가 get 요청으로 데이터를 불러오는데 사용된다면 , useMutation 은 React Query를 이용하여 **서버에 직접 데이터 변경 작업을 요청** 할때 사용. 
`isLoading` 과 `data`는 useQuery와 같이 완료여부와 데이터를 리턴하지만, 
`mutate`는 mutation으로 제공된 함수를 직접만듬

- userMutation애 들어가는 argument
1. `mutationFn` : 데이터를 수정하고, 삭제하는데 사용할 함수. fetch / asxios를 사용하여 Promise를 return하는 함수가 와야합니다.

2. `options`: 의 4가지 소개
  - `onMutate`: promise객체를 return하는 callback함수 mutation함수가 실행되기전에실행되고 mutation함수와 동일한 변수가 제공된다.
  - `onError`:mutation이 실패한 경우 실행할 callback 함수에 해당합니다.
  - `onSettled`:mutation의 성공/실패 여부와 상관없이 실행할 callback 함수에 해당합니다.
  - `onSuccess`:mutation이 성공한 경우 실행할 callback 함수에 해당합니다.
